> [!IMPORTANT]
> This article applies to the **Common APIs**, the Office JavaScript API model that's supported by Office 2013 and later, and can be used to access features such as UI, dialogs, and client settings that are common across multiple types of Office applications. 
> 
> You should use host-specific APIs whenever feasible, and use Common APIs only for scenarios that aren't supported by host-specific APIs. For more detailed information about these two API models, see [Understanding the JavaScript API for Office](understanding-the-javascript-api-for-office.md).
